## 核心实现怎么理解

### 1. 明文密码如何平滑迁移

旧用户只有 `password` 时，登录请求先用常量时间比较验证。验证通过后：生成 `password_hash`、删除明文字段、写临时 JSON 文件、用原子替换覆盖原文件，最后才签发 JWT。验证失败不改数据。

这比批量重置密码更适合存量项目；代价是首次登录需要文件写权限。若原子写入失败，登录会明确失败，避免明文密码被无限期保留。

### 2. 为什么失败不能“优雅兜底”

模型接口失败时返回编造文案或图片，会造成三类问题：用户被误导、成功率统计失真、后续 Spark/Hive 分析无法区分真实模型结果与伪结果。本版本规定：真实生成才是 `success`；DashScope 失败为 `502`；MySQL/Hive 不可用为 `503`。

错误格式固定为：

```json
{
  "status": "unavailable",
  "code": "DATA_UNAVAILABLE",
  "message": "Data services are temporarily unavailable.",
  "request_id": "uuid",
  "retryable": true
}
```

不返回密码、JWT、数据库密码、SQL、API Key 或原始异常堆栈。

### 3. 为什么 Windows 能跑单测

应用导入时不连接 DashScope、MySQL、Hive；MySQL/PyHive 缺失被视为“依赖不可用”，不是 Python 导入失败。pytest 用 Flask test client 和 monkeypatch 模拟超时、5xx、空响应、数据库不可用，因此不调用真实 API，也不消耗额度。

## 秋招八股速记

### JWT 与 Session

- JWT 是服务端签名、客户端携带的令牌；服务端验证签名和过期时间。
- JWT 不等于加密，payload 默认可被解码，不能放密码或敏感资料。
- Session 通常需要服务端存储状态；JWT 更易横向扩展，但撤销、续期和泄露治理更复杂。
- 本项目：使用 `HS256`，密钥放环境变量，过期时间采用 UTC。

### 密码安全

- 不能加密后可逆保存，也不能保存 MD5/SHA256 这类快速哈希。
- 应使用带盐、故意慢的密码哈希，如 Werkzeug PBKDF2、bcrypt 或 Argon2。
- 比较应避免明显的时序差异；旧明文迁移要保证写入原子性。

### HTTP 状态码与幂等性

- `400` 参数错误，`401` 未认证，`403` 已认证但无权限，`409` 资源冲突。
- `502` 上游模型服务异常，`503` 当前依赖不可用。
- `POST /register` 应防止重复用户；生成接口后续可用幂等键避免重复扣费或重复写日志。

### 超时、重试与熔断

- 超时必须有：连接、读、总任务超时；没有超时会耗尽 Web 线程。
- 只对短暂错误重试，如网络错误、429、部分 5xx；不要对参数错误重试。
- 重试使用指数退避和抖动，避免雪崩；熔断在连续失败后快速失败，保护上游。
- 本版本先明确超时和失败；重试/熔断是后续可量化的第二步。

### Mock 与集成测试

- 单元测试验证自身分支，Mock 边界外的网络/数据库。
- 集成测试验证真实组件协作，成本高、依赖环境。
- Mock 不能变成生产运行时的隐藏兜底；测试 Mock 与正式逻辑必须隔离。

### MySQL、Hive、Spark 数据链路

- MySQL 保存在线事务/原始生成日志，适合接口查询。
- HDFS 存放离线文件；Hive 定义表与 SQL 查询；Spark 批量清洗聚合。
- 统计表不能覆盖在线原始日志表，否则会损失可追溯性。第二阶段要明确 raw 表、Hive 明细表和 MySQL 聚合表的唯一职责。
